## 本期概览

本期聚焦**链上代币数据源的构建方法**，核心探讨了如何通过 BSC RPC 直接扫描链上事件，获取新创建代币的完整信息。笔记从 API 局限性出发，转向更底层的链上数据采集方案，体现了从“依赖第三方”到“自主采集”的技术演进思路。

## 核心知识点

1. **链上扫描 vs. 第三方 API**：直接扫描链上代币创建事件（如 `TokenCreated`）比依赖 four.meme 等第三方 API 更全面，不会遗漏代币。
2. **BSC RPC 端点**：使用 `https://bsc-rpc.publicnode.com/` 作为数据源，通过 `eth_getLogs` 获取事件日志。
3. **关键事件**：`TokenCreated` 事件可获取代币地址、创建者、名称、符号等基础信息。
4. **代币状态查询**：通过 `eth_call` 调用 Portal 合约的 `getTokenV5()` 方法，获取 flap 代币的 reserve、supply、price、status、quoteToken 等动态数据。
5. **总供应量获取**：使用 ERC20 标准接口 `totalSupply()` 获取代币总供应量。
6. **数据维度分层**：基础信息（事件）→ 状态信息（合约调用）→ 标准信息（ERC20 接口），形成完整的数据采集链。
