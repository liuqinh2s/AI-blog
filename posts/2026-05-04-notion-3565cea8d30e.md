# 链上代币数据源

## BSC Token Scanner 数据源详解

### 一、核心数据源（每轮扫描必用）

1. BSC RPC (publicnode)
用途: 链上事件监听 + 状态查询

API 端点: [https://bsc-rpc.publicnode.com/](https://bsc-rpc.publicnode.com/)

获取的数据:

数据	来源	说明
TokenCreated 事件	eth_getLogs	four.meme + flap 新代币地址、创建者、名称、符号
代币状态	eth_call	通过 Portal 合约 getTokenV5() 读取 flap 代币的 reserve/supply/price/status/quoteToken
总供应量	eth_call	ERC20 totalSupply()
特点: 免费、无硬性限流、100% 覆盖新币发现

1. four.meme Detail API
用途: four.meme 平台代币详情

API 端点: [https://four.meme/meme-api/v1/private/token/get/v2](https://four.meme/meme-api/v1/private/token/get/v2)

获取的数据:

字段	含义	用途
holderCount	持币地址数	入场筛/淘汰检查/精筛 (bonding curve 阶段)
price	当前价格	定价基准
progress	bonding curve 进度	入场筛/淘汰检查/精筛
totalAmount	总供应量	入场筛 (必须=10亿)
twitterUrl	Twitter 链接	社交验证
telegramUrl	Telegram 链接	社交验证
webUrl	官网链接	社交验证
descr	代币描述	信息展示
raisedAmount	募资额	实力评估
liquidity	流动性	已毕业代币
launchTime	创建时间	币龄计算
限流: ~5 req/s

1. [flap.sh](http://flap.sh/) 页面 SSR
用途: flap 平台代币社交数据（flap 无 Detail API）

API 端点: [https://flap.sh/bnb/{address}](https://flap.sh/bnb/%7Baddress%7D) (Next.js SSR)

获取的数据:

字段	含义	用途
twitter	Twitter 链接	社交验证
telegram	Telegram 链接	社交验证
website	官网链接	社交验证
特点: 从页面 metadata 提取，IPFS CID 获取元数据

1. DexScreener API
用途: 价格/流动性/交易量/涨跌幅/Boost

API 端点: [https://api.dexscreener.com/tokens/v1/bsc/{addresses}](https://api.dexscreener.com/tokens/v1/bsc/%7Baddresses%7D)

获取的数据:

字段	含义	用途
priceUsd	USD 价格	峰值价格追踪
liquidity.usd	流动性 (USD)	淘汰检查/精筛
volume.h24	24h 成交额	精筛参考
volume.h1	1h 成交额	成交额异动检测
txns.h1.buys/sells	1h 买卖笔数	精筛基础标签
txns.h24.buys/sells	24h 买卖笔数	参考
priceChange.m5/h1/h6/h24	涨跌幅	淘汰检查/趋势判断
boosts.active	Boost 数量	项目方推广力度
[baseToken.name/symbol](http://basetoken.name/symbol)	代币名称/符号	信息展示
pairAddress	交易对地址	交易入口
限流: ~300 req/min

1. GeckoTerminal API
用途: 持币数 + K线数据

API 端点: [https://api.geckoterminal.com/api/v2](https://api.geckoterminal.com/api/v2)

获取的数据:

接口	数据	用途
/networks/bsc/tokens/{addr}/info	holders.count	已毕业代币持币数 (链上索引，最准确)
/networks/bsc/pools/{addr}/ohlcv/minute	15min K线 [ts, o, h, l, c, v]	峰值价格修正、过山车检测、横盘整理检测
/networks/bsc/pools/{addr}/ohlcv/minute?aggregate=1	1min K线	假K线检测
限流: ~30 req/min

二、辅助数据源（特定场景使用）
6. BSCScan API (Etherscan V2)
用途: Top 持仓地址 + 开发者行为分析

API 端点: [https://api.etherscan.io/v2/api?chainid=56](https://api.etherscan.io/v2/api?chainid=56)

获取的数据:

接口	数据	用途
tokenholderlist	Top N 持仓地址及余额	精筛后防线 (持仓集中度)
tokentx	代币转账记录	开发者行为分析 (买入/卖出/加池子/撤池子)
getcreator	代币创建者地址	开发者画像
注意: four.meme 未毕业代币不产生标准 ERC20 Transfer，BSCScan 索引不到 holder 列表

限流: ~5 req/s

1. BSCScan 网页爬取
用途: 已毕业代币持币数 (BSCScan API 免费版不支持)

方式: 爬取 [https://bscscan.com/token/{address}](https://bscscan.com/token/%7Baddress%7D) 页面

获取的数据: 持币地址数

使用场景: GeckoTerminal 查不到时的降级备选

三、待启用数据源（已实现未接入）
8. 币安 Web3 Token Dynamic API
用途: 代币动态数据 (一次调用获取多维度数据)

API 端点: [https://web3.binance.com/bapi/defi/v4/public/wallet-direct/buw/wallet/market/token/dynamic/info/ai](https://web3.binance.com/bapi/defi/v4/public/wallet-direct/buw/wallet/market/token/dynamic/info/ai)

获取的数据:

字段	含义	潜在用途
price	价格	-
holders	持币数	-
liquidity	流动性	-
marketCap	市值	-
volume24h	24h 成交额	-
percentChange1h	1h 涨跌幅	短期动量
percentChange24h	24h 涨跌幅	中期趋势
top10HoldersPercentage	Top10 持仓占比	替代 BSCScan，解决 top10 数据缺失
devHoldingPercent	开发者持仓占比	跑路风险评估
smartMoneyHolders	聪明钱持仓人数	正向信号
smartMoneyHoldingPercent	聪明钱持仓占比	高置信度信号
kolHolders	KOL 持仓人数	传播潜力
kolHoldingPercent	KOL 持仓占比	强信号
proHolders	专业交易者人数	非纯散户盘
注意: 币安 Web3 API 禁止美国 IP 访问，仅韩国首尔服务器可用

1. 币安 Web3 聪明钱信号 API
用途: 聪明钱买卖信号

API 端点: [https://web3.binance.com/bapi/defi/v1/public/wallet-direct/buw/wallet/web/signal/smart-money/ai](https://web3.binance.com/bapi/defi/v1/public/wallet-direct/buw/wallet/web/signal/smart-money/ai)

获取的数据:

字段	含义	潜在用途
direction	买入/卖出方向	聪明钱正在买入=强正向信号
smartMoneyCount	聪明钱数量	多个聪明钱同时买入=高置信度
exitRate	退出率	高退出率=风险信号
maxGain	最大收益	历史收益参考
tagEvents	敏感事件标签	风险预警 (如 rug pull 标记)
10. GMGN 聪明钱 API
用途: 聪明钱地址库

获取的数据: BSC 链上聪明钱地址集合及其标签

用途: 匹配代币持仓中的聪明钱地址

四、大盘情绪数据源
11. Ethereum RPC (publicnode)
用途: ETH Gas 大盘指数

获取的数据: eth_feeHistory → gasUsedRatio (0~1，区块填充率)

1. Solana RPC (mainnet-beta)
用途: SOL TPS 大盘指数

获取的数据: getRecentPerformanceSamples → TPS

五、数据源使用优先级
持币数查询优先级
GeckoTerminal > BSCScan 网页爬取 > four.meme Detail API > 队列缓存
数据源	覆盖阶段	说明
GeckoTerminal	已毕业 (DEX)	链上索引，最准确
BSCScan 网页	已毕业 (DEX)	GT 查不到时的备选
four.meme Detail API	Bonding curve	平台内部记账，毕业后返回 0
队列缓存	兜底	上一轮的持币数
六、数据流向总结
链上发现
↓
入场筛 (four.meme Detail API + [flap.sh](http://flap.sh/) 页面 + 链上 totalSupply)
↓
队列存活 → 每 15 分钟淘汰检查
├── 价格/涨跌幅 ← DexScreener
├── 持币数 ← GeckoTerminal / BSCScan / Detail API
├── 流动性 ← DexScreener
└── 进度 ← Detail API / Portal eth_call
↓
精筛 (标签制)
├── 持币数/进度/流动性/K线/买卖数/成交额/波动/趋势/社交 ← 多源汇总
└── 仿盘检测 ← 本地统计 (零 API)
↓
精筛后防线 (BSCScan Top Holders / 币安 Token Dynamic ← 待启用)
↓
推送/交易
Revert
